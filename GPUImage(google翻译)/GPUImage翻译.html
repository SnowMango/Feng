概述

GPUImage框架是一个BSD许可的iOS库，允许您对图像，实时摄像机视频和电影应用GPU加速过滤器和其他效果。与Core Image（iOS 5.0的一部分）相比，GPUImage允许您编写自己的自定义过滤器，支持部署到iOS 4.0，并且具有更简单的界面。然而，它目前缺乏Core Image的一些更高级的功能，例如面部检测。

对于诸如处理图像或实况视频帧的大规模并行操作，GPU相对于CPU具有一些显着的性能优点。在iPhone 4上，简单的图像滤波器在GPU上的执行速度比等效的基于CPU的滤波器快100多倍。

但是，在GPU上运行自定义过滤器需要大量代码来为这些过滤器设置和维护OpenGL ES 2.0渲染目标。我创建了一个示例项目来做这个：

http://www.sunsetlakesoftware.com/2010/10/22/gpu-accelerated-video-processing-mac-and-ios

并发现有很多样板代码我不得不写在其创作。因此，我把这个框架放在一起，封装了处理图像和视频时会遇到的很多常见任务，使得你不需要关心OpenGL ES 2.0基础。

在处理视频时，这个框架比Core Image好一些，在iPhone 4上只需要2.5 ms就可以从相机上传帧，应用伽马滤波器和显示，而使用Core Image的同一操作则为106 ms。基于CPU的处理需要460 ms，使得GPUImage在该硬件上比Core Image快40倍，并且比CPU绑定处理快184倍。在iPhone 4S上，GPUImage对于这种情况只比Core Image快4倍，比CPU绑定的处理快102倍。然而，对于更复杂的操作，如高斯模糊在更大的半径，Core Image目前超过GPUImage。

技术要求

  • OpenGL ES 2.0：使用它的应用程序不会在原始的iPhone，iPhone 3G，第一和第二代iPod touch上运行
  • iOS 4.1作为部署目标（4.0没有电影阅读需要一些扩展）。如果您希望在拍摄静态照片时显示实时视频预览，则需要iOS 4.3作为部署目标。
  • iOS 5.0 SDK来构建
  • 设备必须有相机才能使用相机相关功能（显然）
  • 该框架使用自动引用计数（ARC），但应支持使用ARC和手动引用计数（如果作为子项目添加）的项目，如下所述。对于面向iOS 4.x的手动引用计数应用程序，您需要在应用程序项目的其他链接器标记中添加-fobjc-arc。

一般建筑

GPUImage使用OpenGL ES 2.0着色器来执行图像和视频操作比在CPU限制例程中可以做的更快。但是，它隐藏了在简化的Objective-C接口中与OpenGL ES API交互的复杂性。此接口允许您定义图像和视频的输入源，将过滤器附加到链中，并将生成的处理后的图像或视频发送到屏幕，UIImage或磁盘上的电影。

视频的图像或帧从源对象上传，源对象是GPUImageOutput的子类。这些包括GPUImageVideoCamera（用于来自iOS相机的实时视频），GPUImageStillCamera（用于用相机拍摄照片），GPUImagePicture（用于静止图像）和GPUImageMovie（用于电影）。源对象将静态图像帧上传到OpenGL ES作为纹理，然后将这些纹理移到处理链中的下一个对象。

链中的过滤器和其他后续元素符合GPUImageInput协议，允许它们从链中的上一个链接接收提供或处理的纹理，并对其进行处理。对象在链的更下游一步被认为是目标，并且处理可以通过将多个目标添加到单个输出或过滤器来分支。

例如，从摄像头接收实时视频，将该视频转换为棕褐色调，然后在屏幕上显示视频的应用程序将建立一个链接，如下所示：

GPUImageVideoCamera -> GPUImageSepiaFilter -> GPUImageView

将静态库添加到您的iOS项目

注意：如果你想在Swift项目中使用它，你需要使用“添加这个框架”部分中的步骤，而不是下面的步骤。Swift需要第三方代码的模块。

一旦你有最新的框架源代码，它是相当简单的将它添加到你的应用程序。首先将GPUImage.xcodeproj文件拖动到应用程序的Xcode项目中，以将框架嵌入项目中。接下来，转到应用程序的目标并将GPUImage添加为目标依赖关系。最后，您需要将libGPUImage.a库从GPUImage框架的Products文件夹拖动到应用程序目标中的Link Binary With Libraries构建阶段。

GPUImage需要一些其他框架链接到你的应用程序，所以你需要添加以下链接库在你的应用程序目标：

  • CoreMedia
  • CoreVideo
  • OpenGLES
  • AVFoundation
  • QuartzCore

您还需要找到框架头文件，因此在项目的构建设置中将标题搜索路径设置为从应用程序到GPUImage源目录中的framework /子目录的相对路径。使此标题搜索路径递归。

要在应用程序中使用GPUImage类，只需使用以下内容包含核心框架标题：

#import "GPUImage.h"

注意：如果你尝试使用Interface Builder构建一个接口时遇到错误“未知类GPUImageView在Interface Builder”等等，你可能需要在项目的构建设置中添加-ObjC到你的其他链接器标记。

此外，如果您需要将其部署到iOS 4.x，看起来当前版本的Xcode（4.3）要求您弱连接您的最终应用程序中的核心视频框架或您看到崩溃与消息“没有找到符号：_CVOpenGLESTextureCacheCreate“，当您创建一个存档上传到App Store或临时分发。为此，转到项目的Build Phases选项卡，展开Link Binary With Libraries组，并在列表中找到CoreVideo.framework。将列表最右侧的设置从“必需”更改为“可选”。

此外，这是一个支持ARC的框架，所以如果你想在一个手动引用计数应用程序针对iOS 4.x使用这个，你需要添加-fobjc-arc到你的其他链接器标志。

在命令行中构建静态库

如果您不想将项目作为依赖项包括在应用程序的Xcode项目中，则可以为iOS模拟器或设备构建通用静态库。要做到这一点，运行build.sh在命令行。由此产生的库和头文件将位于build/Release-iphone。您也可以通过改变改变了iOS SDK版本IOSSDK_VER变量build.sh（所有可用版本可以使用找到xcodebuild -showsdks）。

将此作为框架（模块）添加到Mac或iOS项目中

Xcode 6和iOS 8支持使用完整的框架，就像Mac一样，这简化了将它添加到应用程序的过程。要将其添加到应用程序中，我建议将.xcodeproj项目文件拖动到应用程序的项目中（就像在静态库目标中一样）。

对于您的应用程序，转到其目标构建设置，然后选择构建阶段选项卡。在目标依赖关系分组下，在iOS上添加GPUImageFramework（而不是GPUImage，它构建静态库）或在Mac上添加GPUImage。在链接二进制与库部分下，添加GPUImage.framework。

这应该导致GPUImage构建为一个框架。在Xcode 6下，这也将构建为一个模块，这将允许您在Swift项目中使用它。当设置如上，你应该只需要使用

import GPUImage

拉进来。

然后，您需要添加一个新的复制文件构建阶段，将目标设置为框架，并添加GPUImage.framework构建产品。这将允许框架与您的应用程序捆绑（否则，你会看到神秘的“dyld：库未加载：@ rpath / GPUImage.framework / GPUImage”错误执行）。

文档

文档是使用appledoc从标题注释生成的。要构建文档，请切换到Xcode中的“文档”方案。您应该确保“APPLEDOC_PATH”（用户定义的版本设定）指向一个appledoc二进制文件，可用Github上或通过家酿。它还将构建和安装一个.docset文件，您可以使用您最喜欢的文档工具查看。

执行常见任务

过滤实时视频

要从iOS设备的相机中过滤实时视频，您可以使用以下代码：

GPUImageVideoCamera *videoCamera = [[GPUImageVideoCamera alloc] initWithSessionPreset:AVCaptureSessionPreset640x480 cameraPosition:AVCaptureDevicePositionBack];
videoCamera.outputImageOrientation = UIInterfaceOrientationPortrait;

GPUImageFilter *customFilter = [[GPUImageFilter alloc] initWithFragmentShaderFromFile:@"CustomShader"];
GPUImageView *filteredVideoView = [[GPUImageView alloc] initWithFrame:CGRectMake(0.0, 0.0, viewWidth, viewHeight)];

// Add the view somewhere so it's visible

[videoCamera addTarget:customFilter];
[customFilter addTarget:filteredVideoView];

[videoCamera startCameraCapture];

这会设置来自iOS设备的后置摄像头的视频源，使用预设尝试以640x480进行捕获。此视频是在接口处于纵向模式时捕获的，其中横向左侧安装的摄像机需要在显示之前旋转视频帧。使用来自文件CustomShader.fsh的代码的自定义过滤器随后被设置为来自摄像机的视频帧的目标。这些过滤的视频帧最终在UIView子类的帮助下显示在屏幕上，UIView子类可以呈现从该管道得到的过滤的OpenGL ES纹理。

GPUImageView的填充模式可以通过设置其fillMode属性来改变，因此如果源视频的宽高比与视图的宽高比不同，则视频将被拉伸，以黑条为中心或缩放以填充。

对于混合过滤器和其他包含多个图像的过滤器，您可以创建多个输出，并将一个过滤器添加为这两个输出的目标。输出被添加为目标的顺序将影响输入图像被混合或以其他方式处理的顺序。

此外，如果您希望启用麦克风音频捕获来录制电影，您需要将相机的audioEncodingTarget设置为您的电影作家，例如：

videoCamera.audioEncodingTarget = movieWriter;

捕获和过滤静止图片

要捕获和过滤静态照片，您可以使用类似于过滤视频的过程。而不是一个GPUImageVideoCamera，你使用一个GPUImageStillCamera：

stillCamera = [[GPUImageStillCamera alloc] init];
stillCamera.outputImageOrientation = UIInterfaceOrientationPortrait;

filter = [[GPUImageGammaFilter alloc] init];
[stillCamera addTarget:filter];
GPUImageView *filterView = (GPUImageView *)self.view;
[filter addTarget:filterView];

[stillCamera startCameraCapture];

这将为您提供静态相机预览视频的实时过滤Feed。请注意，此预览视频仅在iOS 4.3及更高版本中提供，因此如果您希望拥有此功能，则可能需要将其设置为部署目标。

一旦你想捕获一张照片，你使用一个回调块，如下所示：

[stillCamera capturePhotoProcessedUpToFilter:filter withCompletionHandler:^(UIImage *processedImage, NSError *error){
    NSData *dataForJPEGFile = UIImageJPEGRepresentation(processedImage, 0.8);

    NSArray *paths = NSSearchPathForDirectoriesInDomains(NSDocumentDirectory, NSUserDomainMask, YES);
    NSString *documentsDirectory = [paths objectAtIndex:0];

    NSError *error2 = nil;
    if (![dataForJPEGFile writeToFile:[documentsDirectory stringByAppendingPathComponent:@"FilteredPhoto.jpg"] options:NSAtomicWrite error:&error2])
    {
        return;
    }
}];

上述代码捕获由预览视图中使用的相同过滤器链处理的全尺寸照片，并将该照片作为JPEG保存在应用程序的文档目录中的磁盘。

请注意，由于纹理大小限制，框架目前无法处理旧设备（iPhone 4S，iPad 2或Retina iPad之前的设备）上大于2048像素宽或更大的图像。这意味着，iPhone 4，其相机输出的静态照片大于这个，将无法捕获这样的照片。正在实施平铺机制来解决这个问题。所有其他设备应该能够使用此方法捕获和过滤照片。

处理静止图像

有几种方法来处理静态图像和创建结果。第一种方法是创建静态图像源对象并手动创建过滤器链：

UIImage *inputImage = [UIImage imageNamed:@"Lambeau.jpg"];

GPUImagePicture *stillImageSource = [[GPUImagePicture alloc] initWithImage:inputImage];
GPUImageSepiaFilter *stillImageFilter = [[GPUImageSepiaFilter alloc] init];

[stillImageSource addTarget:stillImageFilter];
[stillImageFilter useNextFrameForImageCapture];
[stillImageSource processImage];

UIImage *currentFilteredVideoFrame = [stillImageFilter imageFromCurrentFramebuffer];

请注意，对于从过滤器手动捕获图像，您需要设置-useNextFrameForImageCapture，以便告知过滤器您以后需要捕获它。默认情况下，GPUImage重用在过滤器内的帧缓冲区以节省内存，所以如果你需要持有一个过滤器的帧缓冲区手动图像捕获，你需要提前知道。

对于要应用于图像的单个过滤器，您只需执行以下操作：

GPUImageSepiaFilter *stillImageFilter2 = [[GPUImageSepiaFilter alloc] init];
UIImage *quickFilteredImage = [stillImageFilter2 imageByFilteringImage:inputImage];

编写自定义过滤器

这个框架相对于Core Image（iOS 5.0）的一个显着优点是能够编写自己的自定义图像和视频处理过滤器。这些过滤器以OpenGL ES 2.0碎片着色器提供，用类似C的OpenGL着色语言编写。

自定义过滤器用代码初始化

GPUImageFilter *customFilter = [[GPUImageFilter alloc] initWithFragmentShaderFromFile:@"CustomShader"];

其中用于片段着色器的扩展名是.fsh。此外，如果您不希望在应用程序包中运送片段着色器，则可以使用-initWithFragmentShaderFromString：initializer将片段着色器提供为字符串。

片段着色器对要在该过滤器阶段渲染的每个像素执行它们的计算。他们使用OpenGL着色语言（GLSL）来做到这一点，它是一种类似C语言的语言，增加了2-D和3-D图形。片段着色器的示例是以下棕褐色滤波器：

varying highp vec2 textureCoordinate;

uniform sampler2D inputImageTexture;

void main()
{
    lowp vec4 textureColor = texture2D(inputImageTexture, textureCoordinate);
    lowp vec4 outputColor;
    outputColor.r = (textureColor.r * 0.393) + (textureColor.g * 0.769) + (textureColor.b * 0.189);
    outputColor.g = (textureColor.r * 0.349) + (textureColor.g * 0.686) + (textureColor.b * 0.168);    
    outputColor.b = (textureColor.r * 0.272) + (textureColor.g * 0.534) + (textureColor.b * 0.131);
    outputColor.a = 1.0;

    gl_FragColor = outputColor;
}

对于在GPUImage框架内可使用的图像滤波器，需要使纹理坐标变化（对于纹理内的当前坐标，归一化为1.0）和inputImageTexture均匀（对于实际输入图像帧纹理）的前两行。

着色器的其余部分抓取传入纹理中该位置处的像素的颜色，以产生棕褐色调的方式操作它，并将该像素颜色写出以用于处理的下一阶段管道。

在Xcode项目中添加片段着色器时需要注意的一点是，Xcode认为它们是源代码文件。要解决这个问题，您需要手动将您的着色器从编译源构建阶段移动到复制Bundle Resources对象，以使着色器包含在您的应用程序包中。

过滤和重新编码电影

电影可以通过GPUImageMovie类加载到框架中，过滤，然后使用GPUImageMovieWriter写出。GPUImageMovieWriter也足够快，可以从iPhone 4的摄像头以640x480实时录制视频，因此可以直接过滤视频源。目前，GPUImageMovieWriter足够快，可以在iPhone 4上录制高达20 FPS的实时720p视频，以及在iPhone 4S（以及新iPad上）录制720p和1080p视频（30 FPS）。

以下是如何加载示例电影，将其传递通过像素滤波器，然后将结果作为480 x 640 h.264电影录制到磁盘的示例：

movieFile = [[GPUImageMovie alloc] initWithURL:sampleURL];
pixellateFilter = [[GPUImagePixellateFilter alloc] init];

[movieFile addTarget:pixellateFilter];

NSString *pathToMovie = [NSHomeDirectory() stringByAppendingPathComponent:@"Documents/Movie.m4v"];
unlink([pathToMovie UTF8String]);
NSURL *movieURL = [NSURL fileURLWithPath:pathToMovie];

movieWriter = [[GPUImageMovieWriter alloc] initWithMovieURL:movieURL size:CGSizeMake(480.0, 640.0)];
[pixellateFilter addTarget:movieWriter];

movieWriter.shouldPassthroughAudio = YES;
movieFile.audioEncodingTarget = movieWriter;
[movieFile enableSynchronizedEncodingUsingMovieWriter:movieWriter];

[movieWriter startRecording];
[movieFile startProcessing];

录制完成后，您需要从过滤器链中删除录像机，并使用以下代码关闭录制：

[pixellateFilter removeTarget:movieWriter];
[movieWriter finishRecording];

在完成关闭之前，电影将无法使用，因此如果在此时间之前中断，录制将会丢失。

与OpenGL ES交互

GPUImage可以通过分别使用其GPUImageTextureOutput和GPUImageTextureInput类来从OpenGL ES导出和导入纹理。这允许您从OpenGL ES场景录制电影，该场景呈现到具有绑定纹理的帧缓冲区对象，或过滤视频或图像，然后将其作为要在场景中显示的纹理馈送到OpenGL ES。

这种方法的一个警告是，这些进程中使用的纹理必须通过共享组或类似的东西在GPUImage的OpenGL ES上下文和任何其他上下文之间共享。

示例应用程序

几个示例应用程序与框架源捆绑在一起。大多数兼容iPhone和iPad类设备。他们试图展示框架的各个方面，并且应该在框架正在开发的时候用作API的最佳例子。这些包括：

SimpleImageFilter

捆绑的JPEG图像在启动时加载到应用程序中，应用滤镜，并将结果呈现到屏幕。此外，此示例显示了两种接收图像，过滤图像并将其保存到磁盘的方法。

SimpleVideoFilter

像素化滤镜应用于实时视频流，使用UISlider控件，可以调整实时视频的像素大小。

SimpleVideoFileFilter

从磁盘加载电影文件，应用非锐化遮罩过滤器，并将过滤的结果重新编码为另一个电影。

MultiViewFilterExample

从单个摄像机馈送，四个视图被填充以应用于摄像机的实时滤波器。一个是直接相机视频，一个是预编程的棕褐色调，两个是基于着色程序的自定义过滤器。

FilterShowcase

这演示了随GPUImage提供的每个过滤器。

基准套件

这用于通过针对CPU限制例程和Core Image进行测试来测试整体框架的性能。涉及静止图像和视频的基准针对所有三个，在应用中显示结果。

CubeExample

这展示了GPUImage与OpenGL ES渲染交互的能力。从相机捕获帧，将一个棕褐色滤镜应用于它们，然后将它们输入纹理以应用于可以用手指旋转的立方体的表面。这个立方体依次被渲染为纹理背景的帧缓冲对象，并且该纹理被反馈到GPUImage中以在渲染到屏幕之前对其应用像素化滤波器。

换句话说，这个应用程序的路径是相机 - >棕褐色调滤波器 - >立方体 - >像素滤波器 - >显示。
